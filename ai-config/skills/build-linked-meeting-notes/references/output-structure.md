# 出力構成とリンク規約

## 目次

- 1. 標準ディレクトリ構成
- 2. ファイル命名規則
- 3. `index.md` 必須セクション
- 4. `topics/*.md` 必須セクション
- 5. 相互リンク規約
- 6. メモ紐づけ規約（任意）

## 1. 標準ディレクトリ構成

会議ごとに1フォルダを作成し、その中にトップページとトピック詳細を置く。

```text
<meeting-folder>/
├── index.md
├── memo.md                  # 任意（ユーザーがメモを渡した場合）
└── topics/
    ├── 01-<topic-slug>.md
    ├── 02-<topic-slug>.md
    └── ...
```

推奨の会議フォルダ名:

- `YYYY-MM-DD-<meeting-slug>`
- 例: `2026-02-22-product-weekly`

## 2. ファイル命名規則

- トピックファイルは先頭に2桁連番を付ける（`01-`, `02-` ...）。
- スラッグは英数字とハイフン中心にする。
- トピックID（表示用）は `T01`, `T02` を使う。
- ファイル名とトピックIDは同じ順序を保つ。

例:

- `topics/01-release-schedule.md` ← `T01`
- `topics/02-risk-review.md` ← `T02`

## 3. `index.md` 必須セクション

最低限、次の順序で配置する。

1. 会議タイトル + 日付
2. 要約
3. 会議情報（任意: 参加者、時間、場所、録画リンク）
4. 関連会議（任意: 前回/次回の議事録リンク）
5. 主要トピック（目次）
6. 決定事項
7. アクションアイテム
8. 関連メモ（任意）

`主要トピック（目次）` の各項目には以下を含める。

- トピックID（`T01`）
- トピック名
- トピックファイルへのリンク
- 1行要約（任意だが推奨）

`関連会議（任意）` を使う場合の例:

```md
- [前回議事録](../2026-02-15-weekly-progress/index.md)
- [次回議事録](../2026-03-01-weekly-progress/index.md)
```

単発会議では `関連会議（任意）` セクションを省略してよい。

## 4. `topics/*.md` 必須セクション

最低限、次を含める。

1. トップページへの戻りリンク
2. トピックID + タイトル
3. トピック要約
4. 議論の詳細
5. 決定 / 保留
6. 関連メモ（任意）
7. 出典（発話者・タイムスタンプ・引用メモ）

推奨アンカー:

- `index.md` の各目次項目: `topic-t01`, `topic-t02` ...
- トピック詳細ページ先頭: `t01`, `t02` ...

## 5. 相互リンク規約

### index -> topic

`index.md` からのリンクは相対パスを使う。

```md
- [T01 リリース日程の確認](./topics/01-release-schedule.md#t01)
```

### topic -> index（戻りリンク）

各トピックファイル先頭に戻りリンクを置く。

```md
[← 目次へ戻る](../index.md#topic-t01)
```

戻り先をトピック項目にすると、ユーザーが一覧から再度辿りやすい。

## 6. メモ紐づけ規約（任意）

ユーザーがラフメモを渡した場合だけ適用する。

- メモ項目IDを `M01`, `M02` ... で採番する。
- `memo.md` を作る場合、各メモ項目にアンカーを付ける（`m01`, `m02`）。
- `index.md` の `関連メモ` セクションに、メモIDと関連トピックの対応を書く。
- 各トピック詳細ページの `関連メモ` で、該当メモへのリンクを置く。

例:

```md
- [M02 予算上限の確認](../memo.md#m02)
```

メモが存在しない場合は `関連メモ` セクションを省略してよい。
