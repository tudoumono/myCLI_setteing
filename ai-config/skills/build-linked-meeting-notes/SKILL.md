# 会議メモ・議事録リンク化

## 概要

会議のトランスクリプトを、読みやすい階層化Markdown（`index.md` + `topics/*.md`）へ変換する。
トップページに日付・要約・目次・決定事項・アクションを置き、各トピック詳細ページからトップページへ戻れる相互リンクを必ず張る。

## 基本方針

- 先に出力構造を固定してから内容を埋める。
- トピックIDを安定させる（`T01`, `T02` など）ことで、後から追記してもリンクを壊さない。
- 曖昧な発言は推測で断定せず、`[要確認]` を付ける。
- メモがある場合はメモID（`M01` など）を付け、関連トピックへリンクする。

## 使うリソース

- `references/output-structure.md`: 出力ディレクトリ構成、命名規則、相互リンク規約を確認するときに読む。
- `references/extraction-guidelines.md`: トピック分割、要約、決定事項・アクション抽出、メモ紐づけの判断基準を確認するときに読む。
- `scripts/check_links.py`: 生成した議事録フォルダ内の相対リンクとアンカー参照を検証するときに使う。
- `scripts/check_placeholders.py`: `{{...}}` の未置換プレースホルダが残っていないか確認するときに使う。
- `assets/templates/index.md`: トップページ（会議サマリー＋目次）の雛形として使う。
- `assets/templates/topic.md`: トピック詳細ページの雛形として使う。
- `assets/templates/memo.md`: ユーザーのラフメモを `M01` 形式で整理し、トピックと紐づけるための雛形として使う。

## ワークフロー

### 1. 入力を確認する

- 会議日付（必須）を確認する。不明ならユーザーに確認する。
- トランスクリプトの形式を確認する（話者名あり/なし、タイムスタンプあり/なし）。
- 追加メモの有無を確認する。ある場合はメモを捨てずにIDを振って残す。
- 出力先を確認する。指定がなければ会議用フォルダを新規作成する。

### 2. トランスクリプトを整形する

- 明らかなノイズ（重複語、音声認識の崩れ）を軽く正規化する。
- 発話の意味を変える編集をしない。
- 発話者やタイムスタンプがある場合は保持する。
- 1つの会話の流れを追える単位で段落化する。

### 3. トピックを抽出してIDを振る

- 議題、論点の切り替わり、決定、宿題（アクション）を手掛かりに分割する。
- `T01` から順番にトピックIDを採番する。
- 各トピックに短いタイトルを付ける（ファイル名に使えるよう簡潔にする）。
- 必要に応じて、トピック内で小見出し（背景/議論/決定/保留）を作る。

### 4. トップページを作成する（`index.md`）

- `assets/templates/index.md` をベースに作成する。
- 先頭に会議日付と会議タイトルを置く。
- 先頭近くに 3〜8 行程度の要約を書く。
- `主要トピック（目次）` に、各トピックファイルへのリンクを並べる。
- `決定事項` と `アクションアイテム` を抽出して記載する。
- `アクションアイテム` には各タスクの実施者（担当）を記載する。トランスクリプト上で不明なら空欄にせず `不明` と書く。
- メモがある場合は `関連メモ（任意）` にメモIDと関連トピックへのリンクを載せる。
- 定例会議や連続会議の場合のみ `関連会議（任意）` に前回/次回の議事録リンクを相対パスで載せる。
- 単発会議では `関連会議（任意）` セクションを削除してよい。

### 4.5. メモページを作成する（`memo.md` / 任意）

- ユーザーがラフメモを渡した場合のみ作成する。
- `assets/templates/memo.md` をベースに作成する。
- メモ項目ごとに `M01`, `M02` のIDを振る。
- 各メモ項目にアンカー（`m01`, `m02`）を付ける。
- 各メモ項目から関連トピック（`topics/*.md`）へ相対リンクを張る。
- 根拠が弱い紐づけは `[要確認]` を付ける。

### 5. トピック詳細ページを作成する（`topics/*.md`）

- `assets/templates/topic.md` をベースに、トピックごとに1ファイル作る。
- ファイル先頭にトップページへの戻りリンクを置く。
- ファイル内に `トピック要約`、`議論の詳細`、`決定/保留`、`関連メモ`、`出典` を入れる。
- トピック内で具体的な作業タスクが出ている場合は `タスク整理（このトピック内 / 任意）` を追加し、実施者を記載する（不明なら `不明`）。
- `出典` には可能なら発話者名とタイムスタンプを残す。
- トップページからのリンク先を安定させるため、トピックIDアンカーを置く。

### 6. 相互リンクを検証する

- 可能ならスクリプトで先に機械検査する。
  - `python3 scripts/check_links.py <meeting-folder>`
  - `python3 scripts/check_placeholders.py <meeting-folder>`
- `index.md` の全トピックリンクが存在するファイルを指しているか確認する。
- 各トピックファイルに `../index.md` への戻りリンクがあるか確認する。
- メモリンクを作った場合は、メモ側とトピック側の双方から辿れるか確認する。
- 壊れたリンクや未確定事項があれば、`[要確認]` として明示する。

## 既存議事録を更新する場合

- 既存のトピックIDを変更しない。
- 内容追加時は既存リンクを壊さないよう、同じファイルへ追記する。
- 新規トピックだけ末尾に追加採番する（例: 既存が `T01`〜`T05` なら新規は `T06`）。
- 要約・決定事項・アクション一覧はトップページで再集約する。

## 品質チェックリスト

- 日付がトップページに明記されている。
- 要約がトップページ先頭にある。
- すべてのトピックが目次に載っている。
- すべてのトピック詳細からトップページへ戻れる。
- 決定事項とアクションが混同されていない。
- アクションアイテムの実施者が空欄になっていない（不明な場合は `不明`）。
- 不明点がある場合、推測で埋めずに `[要確認]` を残している。
- 前回/次回リンクを使う場合、会議フォルダ間の相対パスが正しい。
- `{{...}}` の未置換プレースホルダが残っていない。

## 実行時の出力例（ユーザー依頼の言い換え）

- 「会議の文字起こしから、日付付きの `index.md` とトピック別Markdownを作って」
- 「議事メモをトピックごとに分けて、目次と双方向リンクを付けて」
- 「前回の議事録フォルダに今回分を追記しつつ、アクション一覧を更新して」

## 注意点

- 日本語会議では見出しアンカーの自動生成規則がレンダラー依存になりやすいため、必要に応じて明示アンカーを使う。
- 音声認識の誤変換が疑われる用語は、文脈で補正しつつ元の語を `出典` に残す。
- 機密性の高い会議では、要約に機密情報を過度に再掲しない。
- 前回/次回リンクは任意機能なので、単発会議では無理に入れない。
