# Backlog Management

プロジェクトで保留にしたい案を `BACKLOG.md` に残し、再検討可能な状態で維持するためのスキルです。

## 使うタイミング

- 実装中に「今は着手しないが後で検討したい」案が出た時
- セッション終了前に保留事項を記録したい時
- 保留項目を「完了」へ移動して履歴を整理したい時

## 対象ファイル

- 基本: `<project>/BACKLOG.md`
- ファイルがなければ作成してから追記する

初期テンプレート:

```markdown
# Backlog

保留中のアイデアや、条件付きで将来実装する案を記録する。

## ルール

- 案ごとに「トリガー条件」を明記する（いつ再検討するか）
- 実装したら「完了」へ移動し、対応コミットを記載

## 保留中

（なし）

## 完了

（なし）
```

## 追記フォーマット（保留中）

```markdown
### <タイトル>

- **作成日**: YYYY-MM-DD
- **トリガー**: <再検討条件>
- **概要**: <何をしたいか>
- **次アクション**: <再開時の最初の一手>
- **状態**: 保留
```

## 完了時フォーマット（完了）

```markdown
### <タイトル>

- **完了日**: YYYY-MM-DD
- **対応コミット**: <commit hash or PR>
- **結果**: <要約>
```

## 運用ルール

- 保留項目は必ず「トリガー」を書く
- 1項目1見出し（`###`）で管理する
- 実装後は「保留中」に残さず「完了」へ移動する
- プロジェクト固有でない学びがあれば `sync-knowledge` の適用を提案する

## 次のステップ

保留項目に汎用的な知見が含まれる場合は、`sync-knowledge` でグローバルKBへの反映を検討してください。
