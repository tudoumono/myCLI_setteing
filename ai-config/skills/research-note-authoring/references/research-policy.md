# 調査ポリシー（一般論・最新情報）

一般論をまとめる場合でも、技術情報は変化が速い。特に AWS / LLM / Dify / RAG 実装は最新確認を行う。

## 優先順位

1. 公式ドキュメント（一次情報）
2. 公式ブログ・リリースノート
3. ベンダー提供のサンプルコード/仕様ページ
4. 補助的な解説記事（一次情報で裏取りする）

## ノートへの記載ルール

- 重要な仕様や比較表は、確認日を明記する。
- 日付依存の内容（料金、モデルID、制限値、UI仕様）は「確認時点」を書く。
- 事実と推測を分ける。
  - 事実: ドキュメントに明記された内容
  - 推測: 実運用上の解釈、設計判断、PoC前提
- 参考リンクを残す（最低1つ、可能なら複数）。

## AI議論のまとめ方

- 議論ログをそのまま貼るのではなく、論点ごとに再構成する。
- 結論だけでなく、判断理由とトレードオフを書く。
- 未確認事項は `要確認` として残す。

## 初学者向けの書き方

- 用語を初出で短く説明する。
- 比較は「何が違うか」から書く。
- 先におすすめパターンを示し、その後に例外条件を書く。
